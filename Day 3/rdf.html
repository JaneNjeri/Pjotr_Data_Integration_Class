7 Triples

Triple: Subject - Predicate - Object 

John    - lives in  - Utrecht
Mary    - lives in  - Utrecht
Mary    - has pet   - Mimi

Where does Mimi live? 

Mimi -> pet of -> Mary -> lives in -> Utrecht

But Mimi also is John's pet. Bummer, we miss some information. But we can add it: 

John    - partner of - Mary

Now we know 

Mimi -> pet of -> Mary -> partner of -> John

Mimi may not be John's pet per se, wat we can be sure John knows Mimi if he ever visits Mary.

Data becomes more like 'language'. What about Leo 

Leo     - partner of - John

Let's be more precise 

Leo     - squash partner of - John

and 

Bello   - pet of - Leo

AHA, know we know 

Bello -> pet of -> Leo -> squash partner of -> John

11 SPARQL

SPARQL is not SQL

Let's do an exercise in http://dbpedia.org/snorql/ 

PREFIX owl: <http://www.w3.org/2002/07/owl#>
PREFIX xsd: <http://www.w3.org/2001/XMLSchema#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>
PREFIX foaf: <http://xmlns.com/foaf/0.1/>
PREFIX dc: <http://purl.org/dc/elements/1.1/>
PREFIX : <http://dbpedia.org/resource/>
PREFIX dbo: <http://dbpedia.org/property/>
PREFIX dbpedia: <http://dbpedia.org/>
PREFIX skos: <http://www.w3.org/2004/02/skos/core#>
SELECT ?name ?birth ?description ?person WHERE {
      ?person a dbo:MusicalArtist ;
              dbo:birthPlace :Berlin ;
              dbo:birthDate ?birth ;
              foaf:name ?name ;
              rdfs:comment ?description .
      FILTER (LANG(?description) = 'en') .
} ORDER BY ?name

12.1 URI's

Valid URI's are

  • foaf:name

is equivalent to

  • http://xmlns.com/foaf/0.1/name

So, in our earlier exercise we have to URIify our elements 

@prefix : <http://umcutrecht.nl/mydata/> .

:John :livesIn   :Utrecht
:Mary :livesIn   :Utrecht
:Mary :hasPet    :Mimi
:John :partnerOf :Mary

So :John becomes http://umcutrecht.nl/mydata/John

Valid!! But it would be better to use generic terms, so 

@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix foaf: <http://xmlns.com/foaf/0.1/> .
@prefix relationship: <http://www.perceive.net/schemas/20021119/relationship/#friendOf>
@prefix vcard: <http://www.w3.org/2006/vcard/ns#> .
@prefix : <http://umcutrecht.nl/mydata/> .

:John a foaf:Person
:Mary a foaf:Person
:John vcard:locality :Utrecht
:Mary vcard:locality :Utrecht
:Mary :hasPet :Mimi
:John relationship:spouseOf :Mary

So the expansion for John is

http://umcutrecht.nl/mydata/John http://www.w3.org/1999/02/22-rdf-syntax-ns#type http://xmlns.com/foaf/0.1/Person

https://www.w3.org/TR/vcard-rdf/. The above is valid RDF and can be read into any RDF database (triple store).

12.2 Ontology

Anyone can set up a 'ontology'. We can create name space 'ex':

Some modelling 

(...)
@prefix ex: <http://umcutrecht.nl/mydata/ex> .

ex:Cat rdfs:subClassOf ex:Animal
ex:hasPet rdfs:subPropertyOf ex:livesWith
:Mimi a ex:Cat

Mimi is an cat/animal who lives as a pet with Mary. (where a equals rdf:type)

triples, triples and more triples (ttt or t3?). The genius is that is describes all relationships.
